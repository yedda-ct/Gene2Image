Gene2Image


摘要

空间转录组技术能够同时测得细胞的基因表达与对应的组织图像，为研究分子状态与组织形态的关系提供了数据基础。基因表达描述细胞内部所处的分子状态，组织图像描述细胞呈现的形态；将两者对应起来，有助于我们理解分子变化如何对应可观察的形态，也可能为形态相关的生物标志物提供计算线索。目前从基因表达生成病理图像的方法，有的采用可随任务学习的 RNA 编码器，有的采用预先算好的通路分数，有的采用大规模预训练的基因模型；但在单细胞层面，如何既保留通路结构、又让通路内各基因的贡献随生成目标一起学习，仍缺少专门的设计。本文尝试提出 Gene2Image，一种结构化的通路 RNA 编码器：以固定的通路–基因二值掩码规定哪些连接可以存在，在每条连接上学习一个向量来聚合基因表达、得到通路 token，再用自注意力和一个 CLS token 汇总成 512 维条件，用以指导校正流 U-Net 生成配对的组织图像。其想法是：让已知的生物结构负责哪些基因归到哪条通路，让数据负责每个基因在生成中的贡献大小。为控制内存占用，本文只存有效连接、以稀疏聚合实现这一步，不构造稠密张量。除提出该编码器外，为厘清通路编码究竟起了什么作用，我们在相同的生成器架构与训练设置下分别训练 Gene2Image、GeneFlow 及几个结构变体，重点比较真实通路成员组织相对通路大小匹配的随机成员组织的额外作用，并初步比较固定与可学习的边参数、以及是否引入通路 token 交互。我们还准备了几项模型内部读出，用于描述模型如何使用通路条件，但不将其当作生物因果。实验在三个 Xenium 黑色素瘤样本上进行，用 FID、SSIM 和 PSNR 评价生成质量。

关键词：空间转录组；组织病理图像生成；RNA 条件编码；生物通路先验；校正流


一、引言

空间转录组学能在保留组织空间位置的同时测量基因表达，并能与苏木精–伊红（H&E）染色、DAPI 等组织图像配准，为同一区域同时提供分子与形态信息。基因表达反映细胞内部的分子状态，组织图像反映细胞形态与局部组织结构；将两者联系起来，有助于研究分子变化如何对应可观察的形态表型，也能为形态相关的生物标志物提供计算线索。在肿瘤中，细胞异质性、肿瘤–免疫作用与基质重塑在转录组和形态上均有表现，因此联合建模这两种模态尤为有意义。与从图像预测表达这一常见方向相比，从空间分辨的表达生成对应图像这一反方向仍处于早期阶段：它既要从高维、稀疏、基因间又相关的表达中提取与形态有关的信号，还要处理相似分子状态可能对应多种合理形态这种一对多情况，因此适合采用条件生成模型。

本文关注这一反方向，主要基于以下考虑。第一，它是一种理解性的工具：如果一个模型能从表达生成看起来合理的组织形态，那么考察其内部使用了哪些基因、哪些通路，就有可能提示分子状态与形态之间可能存在的联系，这是一种可以进一步实验检验的假设，而非直接的结论。第二，它可能具有实用价值：配对的表达–图像数据不易获得，条件生成模型至少提供了一种在给定表达下设想对应形态的计算手段。第三，将生成器固定、只更换 RNA 编码方式，可作为一个受控的实验台，用于研究基因表达应如何编码成生成条件这一问题本身。

已有工作采用过多种 RNA 条件表示。RNA-GAN 和 RNA-CDM 用患者层面的批量 RNA 生成组织图像块 [2,3]；GeneFlow 用可学习的注意力 RNA 编码器和条件校正流 U-Net，从单细胞或多细胞表达生成配对图像 [1]；Lohmann 等人先用对比学习将近全转录组表达与图像对齐到共同潜空间，再固定表达编码器、训练 GAN 或扩散生成器 [4]；MUPAD 将 RNA 压缩为 331 个预先算好的通路级富集分数，用于大规模多模态病理生成 [5]；SPATIA 用预训练单细胞基因编码器，并结合细胞邻域、组织尺度等空间上下文 [6]。这些方法各有侧重，但由于数据粒度、附加条件、生成器与评价方式各不相同，难以据此直接判断RNA 编码方式本身起了多大作用。这也是本文希望在统一生成环境下进行受控比较的原因。

本文关注两个问题。第一，如何为这一生成任务设计一个结构化的通路编码器：将已知的通路–基因连接作为固定约束保留下来，同时让通路内各基因的贡献由图像生成目标端到端学习。其直觉是，通路将成百上千个基因组织为几十条有生物含义的组，是一种现成、可读、维度较低的结构；但每条通路中哪个基因对生成图像更重要，事先并不知道，宜交由数据确定。第二，一个模型设计问题：通路编码带来的好处，究竟来自真实通路本身，还是来自稀疏连接、可学习参数、token 交互等一般结构效应？如果真实通路优于按每条通路基因数匹配的随机连接，说明通路成员关系可能提供了生成任务能利用的额外信息；如果两者相近，则只能说在当前数据与设置下未观察到真实通路的额外收益——这既可能是稀疏结构在起作用，也可能是通路 token 瓶颈在起作用，本文的实验并不能将两者区分开。这一问题值得关注，因为已有研究发现，一些知识引导的网络换成随机稀疏连接后性能相近，提示收益可能主要来自稀疏这种归纳偏置，而不一定来自真实生物语义 [11]。

从数据角度也能看到设计动机。本文用到的三个 Xenium 面板基因数差异较大：其中两个面板约有几百个基因，另一个则达到几千个基因。在基因更多的高维面板中，与形态无关的信号也更多，一个不带结构的编码器需自行从头分辨哪些基因有用；而通路这种结构化稀疏，相当于预先将基因分组、限定了连接，我们预期它在高维面板上更可能起用。

为此，本文提出 Gene2Image 结构化通路 RNA 编码器。受通路感知表示学习工作的启发 [7–10]，它用固定二值掩码保留基因到通路的已知拓扑，在每条允许连接上学习一个面向图像生成目标的多维边向量，将基因信号聚合为通路 token，再用自注意力和 CLS token 汇总成统一的生成条件。需要说明，通路 token、掩码映射、Transformer 和 CLS token 这些部件本身在此前工作中已被采用，本文并不声称它们是首次提出；本文在编码器层面的工作，在于将它们围绕“单细胞 RNA 到病理图像生成”这一任务加以组织，使通路结构、通路内可学习权重和校正流条件接口能够一起端到端训练。在此基础上，本文再借助统一生成环境下的结构变体，分析通路编码的作用来源。

本文的主要工作包括：

（1）提出 Gene2Image，一个面向单细胞 RNA 条件病理图像生成的、任务特定的结构化通路编码器。它将固定通路拓扑、可学习的多维边向量、通路 token 交互和 512 维条件接口组合起来，与校正流生成器一起端到端训练。本文采用只存有效边的稀疏实现以控制内存，使这套结构在高维面板上也能训练。

（2）围绕它构建一个比较框架。在相同的生成器架构、条件维度和训练设置下，设置 GeneFlow、随机通路、固定边参数、无 Transformer、稠密连接等对照，重点考察真实通路成员组织相对通路大小匹配的随机分组的额外作用，并初步比较固定与可学习边参数、以及通路 token 交互的影响。各模型分别端到端训练；由于变体之间参数量等并不完全相同，这些比较只作为设计层面的间接证据，而非严格的等参数单因素因果实验。

（3）在三个 Xenium 黑色素瘤样本和统一的校正流骨干上评价该编码器，并给出 CLS 到通路的注意力读出和屏蔽通路 token 的响应作为辅助分析，用于描述模型内部如何使用通路条件，而不将其直接当作生物因果结论。


二、相关工作

2.1 基因表达条件下的病理图像生成

早期工作多采用患者层面的批量 RNA。RNA-GAN 先用变分自编码器压缩表达，再用 GAN 合成组织图像块 [2]；它证明了用表达作为生成条件这一路线可行。RNA-CDM 用级联扩散生成更高分辨率的组织图像，将生成质量向前推进了一步 [3]。它们说明基因表达可以作为生成条件，但使用的是批量 RNA，一个样本的表达混合了大量细胞的信号，难以建立细胞级的分子–形态对应；所生成的也是与该样本整体表达相容的图块，而非某个具体细胞对应的形态。本文关注的是单细胞级、且有空间配对的表达与图像，粒度不同。

空间转录组提供了更细的粒度。GeneFlow 从单细胞或多细胞表达出发，用条件校正流 U-Net 生成 256×256 的 H&E 或多通道图像 [1]；其编码器学习样本相关的低秩基因关系和全局基因注意力，但不显式约束已知的通路–基因邻接，即让模型自行学习基因之间的关系，而不提供现成的通路结构。GeneFlow 与本文任务相同、生成器可以共用，因此本文将其作为主要对照，并在本文统一设置下重新训练其编码分支。Lohmann 等人的两阶段方法先将表达与图像对齐到共同潜空间，再固定表达编码器、训练 StyleGAN 或扩散生成器 [4]，重点在跨模态对齐；其表达编码器在对齐阶段训练完成后即固定，使用的是近全转录组，与本文编码器随生成目标端到端一起学习、且带通路结构不同。近期方法进一步引入通路或预训练：MUPAD 用固定的 331 个通路分数作条件，并用统一的扩散 Transformer 支持多种病理模态生成 [5]；其通路是预先算好的富集分数、训练时不再调整，面向的是大规模多模态生成，而本文让通路内每个基因的贡献随任务学习、且只针对配对图块这一个任务。SPATIA 用预训练 scPRINT 基因编码器，并联合细胞图像、表达、细胞邻域和组织尺度信息，用置信度加权的流匹配建模目标状态的形态分布 [6]，因此它并非仅由 RNA 条件从噪声生成配对图块这同一任务，采用的是大规模预训练加空间上下文的路线，模型规模也大得多。由于这些方法在数据、附加条件、预训练资源和评价特征空间上均不相同，本文只将它们用于定位，不直接比较各自论文中的 FID。

与本文相邻的还有分子扰动条件下的细胞形态生成或编辑。SST-editing 用细胞级空间转录组表达引导免疫荧光图像的 GAN 生成与反演编辑，所做的是在硅片上编辑这类操作 [16]；MorphDiff 以扰动后的 L1000 表达为条件，用潜在扩散生成五通道 Cell Painting 图像，关注的是扰动前后的形态变化 [17]；CellFlux 用流匹配模拟化学或遗传扰动引起的细胞形态变化，通常需要一张控制图像作为输入 [18]。这些工作拓展了基因–形态生成的应用面，但成像模态、条件定义或控制图像输入均与本文不同：本文所做的是由 Xenium 单细胞表达从噪声直接生成配对的 H&E 加 DAPI 图块，没有扰动设定，也不需要控制图像输入。

2.2 通路先验与基因表示学习

GSEA 及其单样本版本用预定义基因集合将基因级表达转为通路级分数 [12,13]，维度更低、语义更清楚，较适合用于可读的下游分析；但其聚合规则通常在下游训练之前就已确定，即通路中的基因如何加权事先固定、不随任务调整。本文借用了按通路分组这一想法，但让组内的权重随图像生成目标一起学习，这是主要区别。另一类方法将生物知识直接写入网络结构：P-NET 按基因、通路和更高层过程构建稀疏层级网络进行前列腺癌预测，以稀疏连接将先验编码进网络 [8]；pmVAE 用可组合的通路模块组织潜变量，使每个模块对应一条通路、得到可解释的表示 [9]；SurvPath 用通路–基因连接限定的稀疏网络得到通路 token，再建模通路之间以及通路与病理图像区域的交互进行生存预测 [10]。这几项工作均说明将通路写入结构在判别或表示任务上是可行、可读的，但它们的目标是预测或表示已有的图像/标签，本文的目标是将通路 token 作为条件生成图像。

其中与本文最接近的是 TOSICA：它用固定的通路–基因二值掩码限制可学习映射，将每条通路表示为多维 token，再与 CLS token 一起经多头自注意力进行细胞类型注释 [7]。本文采用的是同一类结构，但目标与证据设计不同——TOSICA 进行判别式注释，输出的是细胞类型；本文将通路 token 用作生成条件，输出的是图像，并借助统一生成器比较真实通路、随机结构、固定参数和可学习参数，重点不在注释精度而在通路编码起了什么作用。

是否真的需要真实通路仍是一个开放问题。已有研究发现，在一些判别任务上，保持相近稀疏度的随机连接能达到接近真实通路的效果 [11]，提示收益可能主要来自稀疏而非生物语义。受此启发，本文设置 randPath 对照：对每条通路随机抽取与其真实通路相同数量的基因，从而保持通路层面的基因数量。需要强调，该对照并不保持单个基因的连接次数、整体基因覆盖率以及通路间的重叠结构，因此它比较的是"真实通路成员组织"与"通路大小匹配的随机成员组织"，而非对生物通路语义的完全独立隔离；我们以此检验真实成员组织是否提供了超出"大小匹配的随机稀疏"之外的额外信息。本文将这一检验放到生成任务上，也是希望考察在判别任务中观察到的现象，在生成条件下是否同样出现。

2.3 模型内部读出与条件控制

生物医学模型常用注意力、梯度和输入干预来考察模型如何使用分子信息。GeneFlow 用生成速度场对输入表达的梯度估计基因重要性，再结合富集分析 [1]；TOSICA 读取 CLS 到通路 token 的注意力，用以观察哪条通路对某个细胞类型更重要 [7]；SurvPath 分析通路与组织区域的交互 [10]。这些分析能描述模型内部的信息分配，便于我们理解模型在关注什么，但注意力大不一定说明某个通路必要，也不一定对应真实生物机制 [15]。因此本文仅将注意力和通路干预结果当作模型内部读出和条件敏感性，用于描述模型如何组织条件信息，而不解释为生物因果；若要讨论生物因果，仍需真实的扰动实验和外部验证。


三、方法

3.1 问题定义与比较原则

给定配对的单细胞基因表达和组织图像块，本文学习条件分布 p(y|x)，即给定细胞表达、生成与之相容的组织形态。本文特意学习一个条件分布而非确定映射，原因在于相似的分子状态可能对应多种合理形态，一对一的回归会将这些形态平均掉、生成模糊的图像；条件生成模型则允许在同一表达下有多种输出。训练数据记为

    D = { (x_i, y_i) }，i = 1, ..., N，

其中 x_i ∈ R^G 是第 i 个细胞的表达向量，G 是基因数；y_i ∈ [0,1]^(C×H×W) 是以该细胞为中心的图像块。主实验取 H = W = 256、C = 4（H&E 的 RGB 三通道加 DAPI）。对每种编码配置 m，RNA 编码器 E 将表达映射为 512 维条件 c_i = E(x_i)，再由条件速度场 v(z_t, t, c_i) 学习从噪声到图像的变化方向。本文采用统一的 512 维条件接口，是为了让不同编码器仅在如何将表达变为条件这一步不同，其后的生成器完全一致，从而使比较更为清晰。各变体采用相同的 U-Net、条件接口、流路径和训练规则，但各自训练自己的权重；由于它们的参数量、有效连接数和正则参数并不完全一致，本文将结果视为相同生成环境下的设计比较，而非严格的单因素消融。也就是说，本文的比较原则是尽量固定生成环境、只更换编码方式，但我们清楚这并不等于严格控制了所有其他因素，因此结论仅作设计层面的间接证据。

3.2 总体框架

本文方法的核心是 Gene2Image 结构化通路编码器：它用知识库掩码限定允许的通路–基因连接，并让掩码内的参数从生成目标学习。作为对照，本文采用一个不带通路结构的 GeneFlow 式编码器（它学习低秩基因关系和全局基因权重）。这两条路线代表两种思路：一种先给结构、再让数据填入权重；另一种不给结构、让数据自行学习关系。两条路线均输出 512 维条件，接入结构相同、各自训练的校正流 U-Net。数据流如下：

    表达 x → 固定通路掩码 + 可学习边向量 → P 个 48 维通路 token
      → 两层八头 Transformer + CLS 汇总 → 512 维条件
      → 校正流 U-Net → 256×256×4 图像。

选择 48 维的通路 token，是希望每条通路不只有一个标量分数，而是有一个小向量，能够容纳这条通路中不同基因的不同贡献方向；选择 512 维的最终条件，是为了与生成器的条件接口对齐，也为其后的 U-Net 留出足够的信息容量。这些维度对所有编码路线均相同。

3.3 Gene2Image 通路编码器

通路结构。设面板有 G 个基因，筛选后保留 P 条通路，构造二值掩码

    A ∈ {0,1}^(P×G)，A_pg = 1 当且仅当基因 g 属于通路 p。

主实验采用 MSigDB 2023.2 Human Hallmark 基因集 [14]，每条通路与当前 Xenium 面板取交集，只保留至少命中 3 个基因的通路。选择 Hallmark，是因为它条数不多、每条含义清楚、冗余相对较少，适合作为可读的结构先验；要求至少命中 3 个基因，是为了去除在当前面板中几乎没有基因落入的空通路，避免生成没有信息的 token。一个基因可以属于多条通路；未被保留通路覆盖的基因不参与当前条件构建，模型也没有额外的自由基因分支。这是本文有意做出的取舍：将条件限制在通路结构之内，以换取可读性和更强的结构约束，代价是覆盖不到的基因暂时无法利用。掩码的基因顺序与表达矩阵一致。

掩码内聚合。对每条有效连接 (p, g) 学习一个 48 维向量 w_pg。第 i 个细胞第 p 条通路的初始 token 为

    t_(i,p) = Σ_(g: A_pg=1) x_(i,g) · w_pg + b_p，

b_p 为通路偏置。其核心想法是将结构与权重分开：固定掩码决定哪些连接可以存在，是由生物知识给出的骨架；可学习向量决定每条连接在生成中的贡献大小，是数据在该骨架上填入的内容。这样既不丢弃已知的通路组织，也无需事先假定通路中每个基因同等重要。需要说明，这一步将多个基因压缩为一个 48 维 token，并非无损保留全部基因信息，而是在固定通路骨架内保留了基因特异、可随任务调整的贡献。

稀疏实现与复杂度。实现上本文只存有效连接、采用稀疏聚合，不构造稠密张量。具体做法是将所有 A_pg=1 的连接展开为一张边表，前向时对每条边计算 x_(i,g)·w_pg，再用 scatter_add 按通路将同一通路的边求和。这样峰值激活约为 O(N · E · D_token)，其中 N 为细胞数、E 为有效边数、D_token=48；相比之下，若按稠密的 P×G 掩码相乘，峰值约为 O(N · P · G · D_token)。由于真实通路掩码中大部分位置为 0，有效边数 E 远小于 P×G，因此稀疏实现的内存占用要小得多，这也是它能在几千个基因的高维面板上训练的关键。参数量方面，通路掩码嵌入在 Hallmark 设置下约新增 0.36M 参数；完整的 Gene2Image 单细胞编码器加生成条件头约有 42.3M 可训练参数，略少于本文重新训练的 GeneFlow 参考编码器。

交互与汇总。在 P 个通路 token 前加入一个可学习 CLS token，一起经过两层 pre-normalization Transformer；通路没有先后顺序，因此不加位置编码。引入 Transformer，是因为通路之间并不独立，许多通路会共享基因、在生物上也相互关联，让通路 token 彼此交互，可以将这些关系用于条件中；用一个 CLS token 收集全局信息、再从它读出条件，是一种简单统一的汇总方式，也便于后续从 CLS 的注意力观察模型更看重哪些通路。取 CLS 输出，先投影到 256 维，再经两次层归一化与线性映射得到 512 维条件。最后一层 CLS 对各通路的注意力可作为内部读出。

为便于复现，本文将通路编码器的前向过程分为如下几步：

    步骤1 读入细胞表达向量 x，并按固定顺序对齐到通路–基因掩码 A。
    步骤2 对每条有效连接 (p, g)，取该基因表达 x_g 乘以这条边学到的 48 维向量 w_pg。
    步骤3 用 scatter_add 按通路将同一通路的所有边求和，再加通路偏置 b_p，得到 P 个 48 维通路 token。
    步骤4 在这 P 个 token 前拼接一个可学习 CLS token。
    步骤5 经过两层 8 头 pre-norm Transformer，使通路 token 之间以及与 CLS 相互交互。
    步骤6 取 CLS 输出，先投影到 256 维，再经层归一化与线性映射得到 512 维条件 c。
    步骤7 将 c 送入校正流 U-Net 作为生成条件。
3.4 统一的条件校正流生成器与训练目标

所有编码器接入实例化规则相同的条件 U-Net：基础通道 128、每层 2 个残差块、通道倍数 (1,2,2,2)，仅在瓶颈层（最深层，256×256 输入下为 32×32 特征图）施加空间自注意力。将注意力只放在最低分辨率上，是常见的降低算力开销的做法：较高分辨率层用卷积处理局部纹理，最低分辨率层用注意力处理更大范围的结构。时间 t 先经 128 维正弦编码和多层感知机，RNA 条件经另一个多层感知机映射到同一维度，两者相加后传入残差块，通过 scale-shift 归一化调制中间特征。以 scale-shift 的方式注入条件，是让 RNA 条件在每个残差块中缩放和平移特征，而非只在输入处拼接一次，这样条件能持续影响生成的各个阶段。

训练时在高斯噪声 ε 与真实图像 y 之间按正弦路径插值，得到中间状态：

    z_t = sin(πt/2) · y + [1 − sin(πt/2)] · ε，

其中 ε 服从标准高斯。监督的速度目标取该路径关于 t 的导数：

    u_t = (π/2) cos(πt/2) (y − ε)，

损失为 L_flow = E[ ‖v(z_t, t, c_i) − u_t‖² ]，即让网络在每个 t 预测从噪声走向图像应朝哪个方向、走多快。由于 u_t 正是上述插值路径 z_t 关于 t 的精确导数，网络学习的即是这条干净路径上的速度场，训练与推理的边界也保持一致（t=0 为纯噪声、t=1 为图像）。推理从高斯噪声出发，采用 DOPRI5 自适应求解器，步数预算 100；若未积分到 t=1，代码会用有限的额外步补到 t=1，最后将像素截断到 [0,1]。

3.5 GeneFlow 对照编码器

作为无通路结构的对照，本文采用 GeneFlow 公开代码中的单细胞编码分支 [1]，在本文统一的训练设置下重新训练；本文不声称完全复现 GeneFlow 论文的全部实验协议。选择它作为主要对照，是因为它与本文任务相同、能接入同一个校正流 U-Net，代表不给通路结构、让模型自行学习基因关系这一路线，正好与本文的结构化路线形成对照。它先由表达预测秩 K=50 的样本相关低秩因子 U_i、V_i，以残差方式变换表达：x̃_i = x_i + 0.1 (x_i U_i) V_i；低秩加残差的做法，是让模型对每个细胞学习一定的基因之间的样本相关关系，同时以 0.1 的小系数和残差连接保证初始时接近恒等、训练更稳定。再用一个跨样本共享的全局基因注意力向量对各基因作 softmax 加权，相当于学习一套哪些基因整体更重要的全局权重；加权后的表达经 G→512→256 的残差编码、四头特征融合、最终线性投影和 sigmoid 门控，得到 512 维条件。它能从生成任务学习基因关系和基因权重，但不使用外部通路–基因约束，因此基因之间的组织完全依赖数据学习。其可训练参数量约 51.5M，略多于 Gene2Image。主实验采用单细胞路径，不启用 GeneFlow 的多细胞聚合器。

3.6 正则化与模型选择
为约束模型参数并提高训练稳定性，所有模型均采用系数为10的负三次方 的 L1 正则化。对于 Gene2Image、randPath、noTrans 和 noMask，正则项作用于通路–基因边向量；对于 GeneFlow，正则项作用于首个基因到隐藏层的线性投影。PathPrior 的边向量和偏置均被冻结，因此相应正则项不参与梯度更新，也不影响参数优化。
L1 正则化用于抑制贡献较小参数的幅值，鼓励模型形成更紧凑的条件表示。由于各模型的结构、正则化对象及参数数量有所不同，相同系数所产生的有效约束并不完全等价。因此，本文主要将这些实验视为不同完整设计方案之间的比较，而不将性能差异严格归因于单一正则化因素。所有模型均依据验证集上的速度预测均方误差选择检查点，以保证模型选择规则的一致性。

3.7 比较变体

本文设计这几个变体，是希望尽量将通路编码的优势何在拆分为可以分别考察的几个问题：真实通路成员组织（相对大小匹配的随机成员）、可学习 vs 固定、token 交互与汇总、稀疏 vs 稠密。需要提醒，这些变体并非完全等参数的单因素消融，每个变体往往同时改动了不止一处，因此下面标注了每个变体主要隔离或关联的内容。

表 1  各编码设置及对应的研究问题

Gene2Image：完整方法，真实掩码 + 可学习 48 维边向量 + Transformer + CLS。作为其余变体的参照。
GeneFlow：无通路结构的可学习编码器。用于比较两条完整编码路线。
randPath：对每条通路随机抽取与其真实通路相同数量的基因（保持通路层面的基因数量），但不保持单个基因的连接次数、基因覆盖率与通路重叠。它比较的是"真实通路成员组织 vs 通路大小匹配的随机成员组织"，而非对生物通路语义的完全独立隔离。

noTrans：真实掩码 + 可学习边，但去掉 Transformer 和 CLS，改为对通路 token 求平均。
noMask：将掩码换为全 1，保留可学习边和 Transformer。它关联的是稀疏 vs 稠密.


3.8 模型内部读出

训练后，本文准备了两类内部读出，用以描述模型如何使用通路条件。一是取最后一层中多头平均的 CLS 对各通路的注意力，得到每个细胞主要依赖哪几条通路，并可进一步计算注意力熵以观察模型是否集中在少数通路上。二是在相同初始噪声下将某个通路 token 置零或放大，观察生成图像的变化，考察模型对某条通路的敏感程度。它们都只是模型内部读出和条件敏感性，有助于我们理解模型在关注什么，不构成生物因果证据。基于这两类读出的具体分析计划见 4.5。


四、实验设计

4.1 数据与预处理

主实验沿用 GeneFlow 使用过的三个公开 10x Genomics Xenium 人皮肤黑色素瘤样本：C1（Xenium V1 Human Skin Melanoma Base FFPE）、C2（Xeniumranger V1 Human Skin Melanoma Add-on FFPE）、P1（Xenium Prime Human Skin FFPE）[1]，每个样本均包含单细胞表达和配对图像块。选取这三个样本主要基于两点考虑。其一，它们与 GeneFlow 所用样本一致，因此本文在同一任务、同一批数据上与 GeneFlow 式编码器对齐比较时更为公平，也便于他人对照。其二，它们同属黑色素瘤、同一成像平台，疾病与模态相对可控，从而使本文能够将注意力集中在RNA 编码方式本身的差别上，而不必同时处理跨疾病、跨平台的额外变化。训练代码直接读取已处理好的表达矩阵，主训练路径中不再进行归一化或 log1p。图像缩放到 256×256、归一化到 [0,1]，含 H&E RGB 和 DAPI 四个通道。

此外需要说明一点，它也是本文设计实验的一个动机。三个面板的基因数差别不小：C1 和 C2 属于基因数较少的面板，P1 是 Prime 面板、基因数明显更多。基因越多，与形态无关或仅弱相关的信号往往也越多。稀疏的通路结构在此类高维面板上，直觉上更有机会帮助模型只关注应关注的连接、较少受无关信号干扰。因此本文初步预期，结构化稀疏在 P1 这种高维面板上带来的帮助，可能比在 C1、C2 上更明显。

4.2 比较方法与训练设置

主要对照是 GeneFlow 式 RNA 编码器，因其与本文任务相同、能够接入同一个校正流 U-Net [1]；下文表格中将重新训练的这一分支记作GeneFlow。Gene2Image、GeneFlow 及各结构变体采用相同的划分、训练预算、生成器架构和 512 维条件接口，各自端到端训练。MUPAD、SPATIA、Lohmann 等仅用于定位；在没有相同 Xenium 数据和统一生成器复现的情况下，本文不直接比较它们论文中的 FID。训练采用 AdamW（学习率 1e-4、权重衰减 1e-2）、批量 16、梯度裁剪 1.0、混合精度，学习率按余弦退火降至初始的 1%；正式训练脚本将早停耐心设得较大，以训满 50 个 epoch，评价采用验证速度 MSE 最低的检查点。

编码器的稀疏实现代价与参数量已在 3.3 说明，此处不再重复；此外需要指出一点：Gene2Image 相对 GeneFlow 的差别并非通过增加参数量得到，而各变体之间参数量也并不完全相等，因此 4.4 的比较只作为设计层面的对照，而非严格等参数的消融。

为便于复现，本文将几个关键设置固定下来并如实记录。种子固定为 42、43、44，划分、初始化与生成均由种子决定；评价时按固定顺序取最多 5000 个样本，取样顺序不随机，以免引入额外波动。所有变体共用同一个 U-Net、同一套条件接口和同一条流路径，仅在 RNA 编码器部分不同，检查点一律按验证速度 MSE最低来选择。参数量、有效边数等实现统计以统一脚本复核为准。对每个数据集、每个设置，本文均报告三个种子各自的单次结果，再报告均值±标准差，以便读者自行判断稳定性。

4.3 评价指标

采用 FID、SSIM、PSNR。三个主指标仅使用 H&E 的 RGB 三通道，DAPI 不纳入主指标。这三个指标各自衡量的方面不同，结合使用能够更全面地描述生成质量。FID 衡量的是一批生成图与一批真实图在 Inception 特征空间中整体分布是否相似，它对整体的染色风格、纹理统计以及是否存在明显伪影较为敏感，但它属于分布层面，不要求逐样本对齐。SSIM 衡量的是逐样本的结构相似度，将亮度、对比度和结构三部分粗略地综合衡量，对细胞核边界、局部纹理这类结构信息较为敏感。PSNR 衡量的是逐像素的重建误差，对整体的像素级贴合度敏感，但对人眼所关注的结构不一定敏感。

具体实现上，SSIM/PSNR 逐样本计算；FID 采用去掉分类头的 ImageNet Inception-v3 特征（图像缩放到 299×299、从 [0,1] 映射到 [-1,1]）。此实现并非标准 clean-fid，因此 FID 的绝对值仅在本文相同的评价代码下比较，不与其他论文横向比较。每个数据集与设置均报告三个种子的单次结果和均值±标准差；三个种子仅反映划分、初始化和生成的随机性，并非独立病人或生物学重复。

4.5 定性结果与内部分析

定性方面，本文在相同 RNA 条件下并排展示真实图像、GeneFlow、Gene2Image 及主要变体，分析整体染色分布、细胞核边界、局部纹理和空间结构。之所以进行定性对比，是因为几个数值指标各有盲区，直接观察整幅图像能够补充一些数值指标难以体现的差别，例如某个方法是否偏向某种染色、是否使细胞核变得模糊、局部纹理是否过度平滑。。

内部分析部分，本文围绕模型内部究竟如何使用通路条件设计了三项分析，均写在解释脚本 analysis/pathway_interpret.py 中。首先需要说明它们的共同定位：这三项考察的都是模型内部的读出与敏感性，而非真实的生物扰动，也不构成因果证据。

(A) 模型内部主要使用哪些通路。对每个细胞，本文取最后一层中 CLS 对各通路 token 的注意力，作为该细胞的通路使用分布，再计算其注意力熵；熵越低，说明模型将条件信息越集中于少数几条通路，熵越高则说明使用得越分散。随后本文按细胞类型将这些汇总起来，考察每种细胞类型的主导通路是什么、不同细胞类型之间的主导通路差别是否明显。若不同细胞类型确实对应到不同的主导通路，则说明模型并非对所有细胞都使用同一套通路，而是将通路条件用出了细胞类型层面的结构。这一步完全源自模型自身，无需外部标签.

(B) 与已知生物学的一致性。本文将 Gene2Image 内部得到的主导通路，与另一条独立路线得到的通路进行比较：后者是将 GeneFlow 学到的基因重要性做 GSEA 富集，得到一组富集通路。本文计算两边的 top-k 重合率和 Spearman 相关。理由是：Gene2Image 训练时并无任何通路层面的监督，其通路使用完全是为生成图像而自身学得的；若其内部偏好的通路能与一条独立的、基于基因重要性的富集分析相符，则至少说明其通路使用并非随意。

(C) 改动某条通路 token 是否、以及在多大程度上改变生成。为避免"用同一批细胞既选择又验证"的循环，本文把参与该分析的细胞分成不相交的两部分：在选择子集上按 CLS 注意力挑出模型的主导通路，再在与选择子集不相交的干预测量子集上做干预与测量。需说明，该划分只用于避免通路选择与响应测量使用同一批细胞，并不保证这些细胞未参与模型训练。在相同的初始噪声下，将某条通路 token 置零（消融；放大干预作为可选项，默认不运行），测量生成图像相对未改动时的四通道像素 L2 位移；对照组是从全部通路中均匀随机抽取（排除主导通路）的等量通路，而非注意力排名靠后者，以免把"主导 vs 随机"混同为"高注意力 vs 低注意力"。特异性指标 = 主导通路平均位移 / 随机通路平均位移（并给出自助法 95% 置信区间）：若前者明显更大，说明模型对其自身认为重要的通路更为敏感。需再次强调，这仍只是训练后模型的条件敏感性读出，不构成生物因果证据；若要作因果主张，仍需真实分子扰动与外部验证。


五、讨论与局限

尽管本文在统一生成器和训练协议下比较了多种 RNA 编码策略，研究结论仍需结合以下范围加以理解。
首先，本研究基于三个同平台的黑色素瘤样本，为受控比较提供了基础，但样本数量、疾病类型和技术平台仍较为有限。细胞级随机划分也可能保留一定的局部空间相关性；同时，受计算成本限制，部分指标基于最多 5000 个样本计算。因此，当前结果主要反映各方法在统一数据和评价流程下的相对表现，跨病人、跨组织及跨平台的泛化能力仍有待独立队列验证。

其次，本文选择 Hallmark 基因集作为结构先验，主要考虑其规模适中且生物学语义较为清晰，但它并不能覆盖全部基因。当前编码器重点建模被 Hallmark 通路覆盖的基因，尚未为其余基因设置独立表示；此外，基因的多通路归属虽能反映生物过程之间的共享关系，也增加了通路贡献归因的复杂性。未来可进一步比较 Hallmark、Reactome 等不同粒度的通路集合，并探索为未覆盖基因保留补充表示。

再次，randPath、noTrans 和 noMask 等变体主要用于考察真实通路成员组织、token 交互及连接稀疏性等设计因素，但它们并非完全等参数的严格单因素消融，不同模型所接受的 L1 正则规模也有所差异。特别地，randPath 只保持了通路层面的基因数量，并未保持单个基因的连接次数、整体基因覆盖率与通路重叠，因此它比较的是"真实成员组织"与"大小匹配的随机成员组织"，其差异不能完全归因于生物通路语义。因此，这些实验更适合作为多方面的设计证据，而不宜用于对单一组件进行严格归因。PathPrior 的固定权重由训练集统计量构建（不使用验证/测试细胞），本文将其作为探索性参照；后续可采用更严格的参数匹配实验进一步验证。

在评价方面，本文采用统一实现的 FID、SSIM 和 PSNR 比较各模型，但当前 FID 流程与标准 clean-fid 并不完全一致，因此其绝对值不宜直接与其他论文横向比较，仅在本文相同评价代码下用于相对比较。本文按 80/10/10 划分设置了独立的留出测试集：验证集仅用于选择检查点，测试集仅用于报告指标，两者相互独立；各模型在生成时对同一细胞使用同一初始噪声（按细胞标识确定性生成），以支持逐样本的配对比较。注意力、注意力熵和通路 token 干预主要用于描述模型如何利用条件信息，不能替代真实分子扰动实验或外部生物学验证。速度目标为上述干净正弦路径关于 t 的精确导数，不含随时间衰减的额外噪声项，且对所有模型保持一致。

从方法定位看，Gene2Image 与 SPATIA 等大规模预训练方法具有不同侧重。Gene2Image强调较轻量的模型、显式通路结构及受控比较，SPATIA则侧重利用大规模预训练和空间上下文学习更广泛的表示。二者具有潜在互补性。未来工作可进一步将可读的通路接口与预训练基因编码器结合；在独立病人和空间隔离划分下开展跨队列评价；基于多个随机种子和留出集系统验证内部读出；并引入真实分子扰动数据，检验模型条件敏感性与生物学效应之间的关系。

六、结论

本文提出 Gene2Image，一个用于单细胞基因表达条件病理图像生成的结构化通路编码器：以已知通路拓扑约束基因到通路的连接，以可学习边向量和通路 token 使 RNA 条件适应生成目标，输出统一的 512 维条件以驱动校正流 U-Net。实现上采用只存有效边的稀疏聚合，将峰值激活控制在 O(N·E·D_token)，在参数量上也不大于 GeneFlow 参考编码器。在评价这个完整方法的同时，本文还构建了一个比较框架，重点比较真实通路成员组织与通路大小匹配的随机成员组织，并初步比较固定与可学习参数、以及是否存在 token 交互；配套设计了内生性、与 GSEA 富集的一致性、条件敏感性三项内部分析，用于描述模型如何使用通路条件，但均只作为模型内部读出、不作为生物因果。
