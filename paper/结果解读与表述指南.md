# Gene2Image 结果解读与表述指南（claim–evidence 边界）

> 本文件把"每个实验能证明到哪里"固定下来，供填 `【待填】` 结果时按此措辞，避免过度归因。
> 定位一句话：**Gene2Image 是一种新的结构化通路条件编码器；论文核心贡献是——在统一生成骨干下，
> 系统研究"真实成员组织 / 随机结构 / 边参数可学习性 / 通路聚合模块"在 RNA 条件图像生成中的作用。**
> 主结果尚为 `【待填】`，以下为对照与预案，不是已得结论。

## 一、证据边界：能写 / 不能写

| # | 事实 | 可以写 | 不能写 |

## 二、实验—结论对照表（每个实验证明到哪里）

| 实验 | 实际比较内容 | 能支持的结论 |
|---|---|---|
| Gene2Image vs GeneFlow | 结构化通路编码器 vs 自由学习 RNA 编码器（同生成骨干） | 完整设计质量更好/相近/有代价；**不能**单独归因于通路语义 |
| Gene2Image vs randPath | 真实通路成员 vs "每通路大小匹配"的随机成员 | 真实成员组织是否有额外价值；**不能**严格隔离纯生物语义 |
| Gene2Image vs PathPrior | 可学习边向量 vs 固定表达加权边映射代理 | 任务自适应边参数是否优于当前固定代理；**不能**推广为优于 ssGSEA/MUPAD |
| Gene2Image vs noTrans | Transformer+CLS 完整汇总 vs 通路 token 均值池化 | 复杂通路聚合模块是否有额外价值；**不能**只归因于通路间自注意力 |
| Gene2Image vs noMask | 稀疏掩码完整设计 vs 全连接稠密 | 稀疏设计是否更有效/以更少连接达到相近质量；**不能**作稀疏性严格因果证明 |
| RQ4-A | 注意力熵及不同细胞类型的 top 通路 | 注意力是否集中、不同细胞类型是否有不同内部使用模式；**不能**证明通路真实重要 |
| RQ4-B | Gene2Image 注意力排序 vs GeneFlow 重要性 GSEA 排序 | 两条任务内重要性路线是否一致；**不能**称与独立生物学真值一致 |
| RQ4-C | 主导通路 token 置零 vs 随机通路 token 置零 | 模型是否对自身选择的主导通路更敏感；**不能**解释为真实基因扰动/生物因果 |
| Cross-dataset | 源面板训练后向目标面板移植通路权重 | 跨面板直接执行是否可行、方向差异；**不能**证明临床泛化或优于 GeneFlow |

## 三、结果出来后的解读预案（四种情形）

**情形 A — Gene2Image 优于 GeneFlow 且优于 randPath**
> 完整 Gene2Image 设计在统一生成协议下获得更好生成质量；真实通路成员组织相对通路大小匹配的随机成员组织表现出额外价值。
补充：*由于 randPath 未保持基因度数、覆盖率和通路重叠，该差异不能完全归因于生物通路语义。*

**情形 B — Gene2Image 优于 GeneFlow，但与 randPath 相近**（很有价值的机制结果）
> 结构化编码路线优于自由 RNA 编码器，但真实通路成员关系相对大小匹配的随机分组未显示稳定的额外收益。
仅当 Gene2Image 与 randPath **共同**优于 GeneFlow 或 noMask 时，才可加：*结果与"结构化稀疏分组具有作用"的解释一致。* —— 不能仅由 Gene2Image≈randPath 直接断言"稀疏性是原因"。

**情形 C — Gene2Image 与 GeneFlow 相近**
> 显式通路结构在保持相近生成质量的同时，提供了通路层条件接口与内部读出能力，但没有显示稳定的性能优势。
（仍支持"新编码器 + 机制研究"双主线，只是不强调性能提升。）

**情形 D — Gene2Image 低于 GeneFlow**
> 显式通路瓶颈在当前设置下带来生成质量代价，说明增强结构可读性与保留全部任务信息之间存在权衡。
（若 randPath/PathPrior 等结果有规律，机制研究仍有价值；但摘要不能再把 Gene2Image 写成性能改进方法。）

## 四、可直接用于论文的实验总结段 + 定位句

**实验总结段（可放实验设置/摘要邻近）：**
> 本研究在三个配对的 Xenium 黑色素瘤数据集上，采用统一的 512 维条件接口和校正流 U-Net，对六种 RNA 条件编码设置进行了受控比较。主实验比较结构化通路编码器 Gene2Image 与无显式通路约束的 GeneFlow 式编码器；进一步通过 randPath、PathPrior、noTrans 和 noMask，分别考察真实通路成员组织相对于通路大小匹配随机成员组织的增量作用、任务自适应边参数相对于固定映射代理的作用、完整 Transformer 汇总模块的作用，以及稀疏掩码设计相对于稠密连接的表现。所有设置在每个数据集上使用三个种子独立训练，并在细胞级留出测试集上以统一实现的 FID、SSIM 和 PSNR 评价 RGB 生成质量。此外，本文通过注意力分布、跨方法通路排序一致性和通路 token 消融分析模型内部如何利用通路条件。上述结果提供的是统一生成环境下的模型设计与条件敏感性证据，不构成跨患者临床泛化或生物因果证明。

**定位句：**
> Gene2Image 是一种新的结构化通路条件编码器；论文的核心贡献是利用统一生成骨干系统研究真实成员组织、随机结构、边参数可学习性和通路聚合模块在 RNA 条件图像生成中的作用。

## 五、正式结果可信前的 3 个运行门槛（硬性）

跑完后，正式结果被接受前必须同时满足：
1. **DOPRI5 `dt_floor = 0`**（grep 日志 `DOPRI5_DIAGNOSTICS`）；
2. **没有 `UNDER_INTEGRATION_FALLBACK`**（同一日志字段 `under_integration_fallback=0`）；
3. **没有任何评价样本因 NaN 被过滤**——即每个 run 的 `evaluation_summary.json` 中 `n_ssim_used == n_psnr_used == total_samples`；若不相等，日志会有"Dropped … non-finite"告警，须查因，不能悄悄用剩余样本报均值。

`rejected > 0` 属正常（自适应求解器健康表现），不算失败。
