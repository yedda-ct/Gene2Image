## 1. 通路瓶颈编码器 模块 A/B/C（src/pathway_encoder.py）

**作用**：Replaces GeneFlow's unstructured RNA encoder with a biologically-structured, learnable pathway bottleneck: gene expression -> P pathway tokens via a fixed sparse mask with per-edge learnable weights (A), pathway-token self-attention with a CLS token (B), CLS pooling + head to the 512-d conditioning vector (C). Rectified-flow+UNet backbone and its 512-d interface are untouched, so quality differences are attributable to this encoder. Three orthogonal ablation switches (mask / learnable-weights / transformer) are all realized here.

- Eq.(1) faithfulness: the code uses T.scatter_add_(1, idx, contrib) with idx = edge_p expanded to [N,E,d_token] (lines 111-112), which is numerically equivalent to the paper/Algorithm-1 index_add_(1, edge_p, C). Safe to write 'scatter_add (equivalently index_add_)' as the manuscript already does.
- l1_penalty() returns the raw unweighted ||W||_1 = self.W.abs().sum() (line 118). The λ=1e-3 is NOT applied here — do not claim the encoder scales the penalty; the trainer multiplies it into L = L_flow + λ||W||_1.
- The PathPrior ssGSEA weighting is deliberately NOT a plain scalar broadcast: each frozen scalar multiplies a fixed seed-1234567 (line 90), zero-mean (92), unit-norm (93) d_token profile. The extensive code comment (80-89) and manuscript lines 518-522 explain that a constant token would be centered out by the pre-norm LayerNorm, making conditioning input-independent and breaking the RQ3 ablation. State this as a design decision, not a hack.
- The exact seed value 1234567 (line 90) appears only in code; the paper says 'seed-fixed'. You may cite 1234567 as an implementation detail but it is not currently in main.tex.
- The returned CLS→pathway attention is head-AVERAGED (average_attn_weights=True, line 149) and taken from the LAST layer only (need_weights gated at line 192), and drops the CLS self-column via [:,0,1:] to give exactly [N,P] (line 199). Multi returns [B,C_max,P] (356).
- noTrans's get_pathway_attention returns a uniform 1/P vector (line 202), consistent with the paper's 'CLS reduces to a mean over pathway tokens' — attention is not informative in that variant.
- Bias b_p is per-pathway [P,d_token], zero-initialized (line 75), added AFTER the scatter (line 113); in PathPrior it is frozen at that zero init (line 101), matching main.tex:508-509.
- Defaults verified in-file (do not need \TBD): d_token=48, n_layers/L=2, n_heads/H=8, d_cell/D_c=256, output_dim=512, dropout=0.1 — consistent across PathwayMaskEmbedding (45), PathwayTransformer (160-161) and both entry classes (221-223, 325-327).
- forward(x, mask=None) ignores the mask argument (line 237/339 comment) — the pathway mask is injected at construction. The preserved signature is what makes it a drop-in replacement for GeneFlow's RNAEncoder / MultiCellRNAEncoder; do not describe the runtime mask arg as functional.
- edge_p, edge_g and mask are registered buffers (saved, not trained; lines 68-70). The dense P×G×d_token weight tensor is never built — this backs the 'lightweight / ~0.36M-param embedding' claim (main.tex:408), though the 0.36M figure itself is computed outside this file.

## 2. GeneFlow 原始 RNA 编码器 + encoder_type 分支（src/single_model.py, multi_model.py）

**作用**：These two files hold the ORIGINAL GeneFlow RNA encoders (unstructured, gene-indexed) that the paper describes as the baseline in Related Work, plus the `encoder_type` switch that swaps in the learnable pathway bottleneck while keeping the identical 512-d UNet conditioning interface, so any quality delta is attributable solely to the encoder.

- `RNAEncoder`（`src/single_model.py:38`）— Original single-cell GeneFlow RNA encoder (unstructured, gene-indexed). __init__ signature lines 42-49 with all ablation flags; default output_dim=128 but instantiated at 512 by the model.
